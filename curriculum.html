Curriculum

Fundamentals first, then the internals that make Spark fast.

Each module is designed around a small dataset, a short Spark snippet, and an interactive visualization target.

01

Spark Foundations

Cluster roles, SparkSession, jobs, stages, tasks, and why distributed execution matters.

spark = SparkSession.builder.appName("orders").getOrCreate()
df = spark.read.csv("orders.csv", header=True)
  • Spark driver, executors, and cluster manager
  • Transformations vs actions
  • Jobs, stages, tasks, and partitions
  • Local mode vs cluster mode
02

DataFrames and Spark SQL

The main API students should master first: schemas, columns, filtering, aggregation, and SQL.

orders.where("amount >= 50")
  .groupBy("city")
  .agg(sum("amount").alias("revenue"))
  • Schema inference and explicit schemas
  • select, filter, withColumn
  • Aggregations and window functions
  • SQL views and query plans
03

RDD Mental Model

Understand lineage, immutability, partition-level work, and the lower-level model beneath DataFrames.

rdd = sc.textFile("logs")
errors = rdd.filter(lambda line: "ERROR" in line)
errors.count()
  • RDD creation and lineage
  • Narrow and wide dependencies
  • Pair RDDs and key-value operations
  • When RDDs still make sense
04

Execution Internals

See how Spark turns code into logical plans, optimized plans, physical plans, and tasks.

orders.groupBy("city").count().explain("formatted")
  • Lazy evaluation and DAGs
  • Catalyst optimizer
  • Tungsten execution engine
  • Adaptive Query Execution
05

Data Engineering Patterns

Read, transform, validate, and write reliable datasets using production-friendly patterns.

clean.write
  .partitionBy("event_date")
  .mode("overwrite")
  .parquet("s3://lake/silver/orders")
  • CSV, JSON, Parquet, ORC, and Delta-style tables
  • Partitioning and file layout
  • Deduplication and data quality checks
  • Incremental batch pipelines
06

Performance and Scale

Learn the causes of slow Spark jobs: shuffles, skew, spills, bad joins, tiny files, and poor partitioning.

spark.conf.set("spark.sql.adaptive.enabled", "true")
large.join(broadcast(dim), "product_id")
  • Shuffle cost and stage boundaries
  • Broadcast, sort-merge, and shuffle hash joins
  • Caching, persistence, and checkpointing
  • Skew handling and partition tuning
